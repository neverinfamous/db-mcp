# v1.1.1 - Docker Workflow Fix

## Highlights

- 🐳 Fixed Docker CI build failure and updated deprecated GitHub Actions to Node 24
- 🔄 Centralized version management — adapters now import from `src/version.ts`
- 🛡️ Hardened prompt handler, transaction validation, and index table checks

## Fixed

- **Docker Build**: Adjusted Docker/CI build configuration so `tsup` receives the correct inputs
- **Docker Workflow**: All `docker/setup-buildx-action` refs v3 → v4 (Node 20 → 24)
- **Docker Workflow**: `peter-evans/dockerhub-description` broken SHA → v5 (`1b9a80c`)
- **Prompt Handler**: Optional `args` with `?? {}` default prevents crash on SDK `undefined`
- **Transaction Methods**: `ValidationError` with `INVALID_SAVEPOINT_NAME` code replaces generic `Error`
- **Index Tools**: Centralized `validateTableExists()` replaces inline `sqlite_master` checks

## Changed

- **Version Management**: Both adapters import `VERSION` from `src/version.ts` — only `package.json` needs updating on bumps
- **Version Test**: Uses `VERSION` constant instead of hardcoded string
